### WHAT IS VIBE CODING? (0:45 - 2:00)

"Before we start building, let me explain what vibe coding actually is — because there's a lot of confusion out there.

Vibe coding is a term coined by Andrej Karpathy, one of the co-founders of OpenAI. He described it as 'forgetting that code even exists' and 'giving in to the vibes.' What that means in practice is this: instead of typing code in a programming language, you describe what you want in plain English, and AI builds it for you.

Think about it like this. Traditional coding is like learning to speak French before you can order food in Paris. Vibe coding is like having a translator who speaks perfect French — you just tell them what you want, and they handle the rest.

Now, I want to be clear about something important. Vibe coding isn't magic. It's not going to make you a senior software engineer overnight. The code still exists. The complexity is still there. But here's what changes: you don't need to memorize syntax, debug semicolons, or spend months learning frameworks before you can build something useful.

What you DO need is the ability to describe what you want clearly. To think through problems logically. To know what good looks like even if you can't write the code yourself. Those skills? You already have them. You've been using them your whole life.

And that's why this moment is so important. For the first time in history, the barrier to building software isn't technical knowledge — it's imagination and clarity. If you can describe it, you can build it."

### LIVE BUILD: EXPENSE TRACKER (3:30 - 7:30)

"I'm going to build a simple expense tracker right now, and I'm going to narrate my thought process so you can follow along. By the end of this section, you'll have seen exactly how vibe coding works in practice.

**Step 1: Setting up the project**

I open Lovable and click 'New Project.' It asks me what I want to build. Here's exactly what I type:

'Create a personal expense tracker app. I want to be able to add expenses with a description, amount, and category. I want to see a list of all my expenses with the date I added them. I want to see my total spending. Use a modern, clean design with a dark theme and green accents. Make it mobile-responsive.'

That's it. I hit enter, and Lovable starts working. I can see it thinking, breaking down my request, planning the structure. This takes about 30 seconds.

**Step 2: First iteration**

Lovable generates the first version. I can see a preview immediately. It's... actually pretty good. There's a form to add expenses, a list showing my entries, and a total at the top. The dark theme looks clean. The green accents are there.

But I want to make some changes. This is where vibe coding gets powerful — iterative refinement.

I type: 'Add the ability to delete expenses. Also, add a filter so I can view expenses by category. And show the total for each category, not just the overall total.'

Lovable updates the app. Now I have delete buttons, category filters, and category totals. This took about 45 seconds.

**Step 3: Polish and refinement**

The app works, but I want it to feel more polished. I type:

'Add some animations when adding and deleting expenses. Make the numbers animate when they change. Add a simple chart showing spending by category over time.'

Lovable adds smooth animations, number transitions, and a simple bar chart. The app is starting to feel like a real product.

**Step 4: Data persistence**

Right now, if I refresh the page, my data disappears. I need the app to remember my expenses. I type:

'Save the expenses to local storage so they persist when I refresh the page. Also add a button to export all my data as a CSV file.'

Lovable implements local storage and adds an export feature. Now my data survives page refreshes, and I can download a spreadsheet of my expenses.

**Step 5: Final touches**

I spend a few more minutes refining. I ask for a better empty state when there are no expenses. I ask for confirmation before deleting. I ask for better mobile styling. Each request takes 30-60 seconds to implement.

**The result:**

Twenty-eight minutes after I started, I have a fully functional expense tracker. It has:

- Delete expenses with confirmation

- View total spending and category breakdowns
- Visual chart of spending over time

### COMMON BEGINNER MISTAKES (9:00 - 10:15)

"Before you go off and start building, let me save you some frustration. Here are the mistakes I see beginners make over and over:

**Mistake #1: Vague prompts**

Saying 'build me a social network' is like walking into a restaurant and saying 'make me food.' You might get something edible, but it probably won't be what you wanted. Be specific. Describe features. Mention design preferences. Set constraints.

**Mistake #2: Trying to build everything at once**

Start simple. Get a basic version working. Then add features one by one. It's much easier to iterate on something that works than to debug a complex mess that never ran.

**Mistake #3: Not testing as you go**

Every time Lovable generates an update, test it immediately. Click every button. Try edge cases. The sooner you catch problems, the easier they are to fix.

**Mistake #4: Giving up too early**

Sometimes the AI doesn't get it right on the first try. That's normal. Describe what went wrong and ask it to fix it. Vibe coding is a conversation, not a one-shot deal.

**Mistake #5: Not reading what the AI generates**

Even if you don't understand code, read the explanations Lovable provides. Over time, you'll start to recognize patterns. You'll understand more than you think.

Avoid these mistakes, and you'll be building working applications in your first week."
